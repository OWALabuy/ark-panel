# 体验类小功能首版设计决定

日期：2026-07-16

实现状态：首批范围已于 2026-07-16 完成；会话状态区与中英文界面于 2026-07-17 作为独立增量完成；第七节中的其余候选项仍未排期。

## 结论

首批体验工作收敛为六项：**可切换主题与具名强调色**、**桌面端可收起的双侧栏 rail**、**按 agent 的自定义头像**、**网页标题反映会话与 agent**、**按会话记录的后台生成完成提醒**，以及承载偏好项的**设置抽屉与阅读字号**。多语言和键盘快捷键保留为候选，不进入当前排期。

本决定固定首版范围、状态归属和安全边界。实现不得再以“通常只有一个 run”、客户端图片处理可信或只有桌面布局为前提。

## 一、主题与外观

### 首批主题

- `system`：跟随操作系统明暗偏好。
- `light`：内置浅色主题。
- `dark`：内置暗色主题。
- `gruvbox-dark-medium`。
- `gruvbox-light-medium`。

Gruvbox hard/soft 变体和开放式配色盘留作后续。实现时使用 Gruvbox 的规范色值，但这只是色值核对，不再改变首批主题数量。

强调色选项为 `default`、`blue`、`green`、`red`、`yellow`、`magenta`、`cyan`。用户保存的是具名选择；每套主题分别定义对应色值，因此切换主题时保留强调色语义，而不是沿用旧主题的十六进制颜色。

### 主题机制

- 使用 `data-theme` 和 CSS 自定义属性，不在组件规则中散落主题颜色。
- token 至少覆盖页面、面板、浮层、正文、弱化文字、边框、危险状态、强调色、强调色前景、柔和强调色、焦点环、阴影、代码块、语法高亮、KaTeX 错误、滚动条和原生表单。
- 每套主题必须同时定义 `accent`、`accent-soft`、`on-accent` 和焦点色，不能假定任意强调色在任意背景上都有足够对比度。
- 设置 `color-scheme`，使原生控件和滚动条跟随明暗主题。
- 服务端设置是权威值；浏览器缓存最近一次主题并在首屏尽早应用，登录后再与服务端同步，避免暗色用户刷新时闪白。

主题和强调色为账户级、跨设备设置。

## 二、边栏收起与 rail

桌面端把现有 **agent 栏和会话栏整体收起**为约 60px 的竖直 rail；展开后恢复现有双栏结构。收起状态为设备本地设置。

rail 自上而下保留：

1. 品牌 logo 和明确的“打开边栏”按钮。
2. 在 `activeAgent` 下新建会话。
3. 搜索会话。
4. 最近会话 flyout：只列当前 agent 最近更新的 10 条会话。
5. 齿轮：直接打开设置抽屉，不先弹第二层账户菜单。
6. 底部当前 agent 头像：点击打开 agent 列表并切换 `activeAgent`。

切换 agent 后关闭当前会话，清空会话级选择状态并展示新 agent 的会话列表。头像在这里表示当前 agent；退出登录放在设置抽屉底部，不与 agent 切换混用。

必须提供可聚焦、有可访问名称的展开按钮。点击 rail 非按钮空白处可以辅助展开，但不能成为唯一入口，也不能依赖 hover 才能发现。flyout 支持点击外部和 `Escape` 关闭，打开与关闭时正确转移/恢复焦点。

移动端不使用 rail，继续采用当前全屏分层导航；设置抽屉在小屏使用接近全宽的布局。

## 三、身份识别与头像

### 网页标题

- 普通会话：`<会话名> - <agent label>`。
- 已选 agent、尚无会话或未命名的新会话：`新会话 - <agent label>`。
- 登录页、尚无 agent：`ark-panel`。
- 归档会话沿用普通会话格式。
- 通知数量由统一的标题生成函数加在最前面，例如 `(2) 会话名 - Claude`，不得由多个事件处理器分别改写 `document.title`。

### 自定义头像

- 每个 allowlist 内的 agent 可保存一个账户级头像，跨设备共享。
- 前端提供 1:1 裁剪预览；默认头像为 agent label 的首字。
- 原始上传最大 5 MiB，只接受 PNG、JPEG、WebP；不支持 SVG、GIF 或动图。
- 服务端不信任扩展名、`Content-Type` 或 canvas 输出，必须检查实际格式和像素尺寸；输入最大 4096×4096。
- 服务端统一重编码并保存为 256×256 WebP，写入 `PANEL_DATA_DIR` 时使用路径边界检查和原子替换。
- 支持删除自定义头像并恢复默认头像。头像响应使用正确的媒体类型、`nosniff` 和可验证缓存策略。

logo/favicon 仍由用户单独制作，不属于本批实现范围。

## 四、后台生成完成提醒

提醒建立在持久 run 和跨会话并发之上。状态模型是设备本地的未读 map，以会话 ID 为 key，并保存 `agentId` 和最后一种终态（`completed` / `failed`）；不能退化为单个全局布尔值。

- 非当前会话生成成功：加入未读集合，并在会话列表显示小红点。
- 当前 agent 以外的会话有未读时，在展开态 agent 项和 rail 的 agent 切换器中显示聚合标记，切换过去后仍由具体会话项指示来源。
- 当前会话仅在页面可见且用户正在查看时视为已读；页面隐藏时完成仍应加入未读集合。
- 生成失败也加入未读集合，并使用区别于成功的错误样式。
- 用户主动停止不产生通知。
- 页面隐藏时，标题显示未读会话数量；数量是未读会话数，而不是完成消息数。
- 页面重新可见不能直接清空通知。只有成功打开对应会话并呈现最新状态后，才清除该会话的未读标记。
- 未读集合持久化到 `localStorage`，刷新后保留，但不跨浏览器或设备同步。
- 不使用浏览器 Notification API，不申请系统通知权限。

同一会话重复完成只占一个未读项；成功和失败状态可记录最后一种终态，用于选择列表标记样式。

## 五、阅读字号

- 设置抽屉提供 85%–130% 的字号滑块，默认 100%，步进 5%，并提供“恢复默认”。
- 只缩放阅读内容：用户/AI 消息正文、Markdown、代码块、工具输入输出和数学公式。
- 导航、按钮、输入框、设置界面及总体布局尺寸不跟随缩放。
- 字号与对应行高一起按比例调整，但不增加独立“密度”设置。
- 字号为设备本地设置，存入 `localStorage`；非法或超范围值回退到 100%。
- 工具输入/输出继续默认折叠。

## 六、设置抽屉与存储

点击齿轮直接打开设置抽屉。抽屉提供“外观”“阅读”“会话”和“Agent 头像”区域，退出登录作为底部独立操作，不建立只有退出按钮的“账户”分区。抽屉支持关闭按钮、`Escape`、点击遮罩关闭，并在关闭后把焦点还给触发按钮。

### 存储归属

- 账户级、服务端持久化：主题、强调色、会话状态区开关、agent 头像。
- 设备本地：阅读字号、双侧栏收起状态、后台完成未读集合。
- 服务端设置写入 `PANEL_DATA_DIR`，采用严格 schema、大小限制、权限收紧和原子写入。未知字段拒绝，不静默保存。
- 浏览器可镜像最近主题用于首屏，但登录后的服务端值始终覆盖镜像。

### 设置接口

`GET /api/v1/settings` 的响应 `data` 为：

```json
{
  "version": 1,
  "appearance": {
    "theme": "system",
    "accent": "default"
  },
  "conversation": {
    "showStatus": true
  }
}
```

`PATCH /api/v1/settings` 只接受上述 allowlist 内字段并执行局部更新。首版是单用户服务，跨标签页并发保存采用 last-write-wins，不引入 revision；每次写入仍必须原子完成。

头像接口：

- `GET /api/v1/agents/:id/avatar`：读取自定义头像；没有自定义头像时返回 404，由前端显示默认首字头像。
- `PUT /api/v1/agents/:id/avatar`：独立的二进制上传请求，不复用只处理 JSON 的通用前端 API helper。
- `DELETE /api/v1/agents/:id/avatar`：恢复默认头像。

三个头像操作都必须要求登录；PUT/DELETE 继续接受同源与 CSRF 校验，并校验 agent ID 属于配置 allowlist。

### 会话状态区（2026-07-17 已实现）

会话标题下方有独立于生成生命周期副标题的紧凑状态区，展示显式模型 override、思考/reasoning 设置、当前权威 transcript 的上下文安全预算估算和最近活跃时间。没有 override 的面板会话只写“默认模型”，只读来源写“来源默认模型”，不猜测无法证明的模型名称。

上下文百分比来自面板的 `utf8-bytes-upper-bound-v2` 保守预算，不冒充厂商 token 计数或模型 context window；70% 起警告、90% 起危险提示。设置抽屉中的“显示会话状态”开关保存为账户级服务端字段 `conversation.showStatus`，默认开启并跨终端生效。旧设置文件缺少该字段时按开启读取，下一次更新时写入完整 schema；浏览器本地存储不作为该开关的权威来源。

### 会话 project 快捷分组（2026-07-17 已优化）

标题栏和会话列表 hover 操作共用一个可访问的“移动到分组”菜单。菜单目录汇总当前 agent 的正常与归档会话但排除隐藏来源；选择已有项目或“未分组”立即保存，“新建分组”在同一弹层中行内完成。名称按大小写不敏感方式匹配已有项目并沿用现存拼写，避免 `Panel` / `panel` 重复组。

菜单支持方向键、Home/End、分层 Escape、点击外部关闭和焦点恢复。此次只改善高频分配路径，不加入拖拽、分组重命名/删除、颜色、图标、手工排序或独立项目管理页；现有置顶顺序、搜索和设备本地折叠状态不变。

## 七、候选但未排期

| 方向 | 状态 | 决定 |
|---|---|---|
| 多语言 | 2026-07-17 已实现 | 首批支持简体中文与英文；语义键字典保持键和占位符一致，语言为账户级服务端设置，旧配置默认中文。服务端错误码由前端映射，未知错误不把中文正文直接混入英文界面。 |
| 键盘快捷键 / 命令面板 | 候选、未排期 | 未来实现时必须可在设置中关闭，并允许改键，以避免与 Vimium 冲突。 |
| 开放式配色盘 | 候选、未排期 | 首批使用具名强调色。 |
| Gruvbox hard/soft | 候选、未排期 | token 体系稳定后可低成本补充。 |

UI 内修改密码仍不做，继续使用 CLI `npm run password-hash`；这不影响未来重新评估。

## 实现批次与验收边界

1. 把全部视觉颜色迁移到语义 token，完成五种首批主题、强调色、首屏应用和标题生成。
2. 完成字号、本地偏好与设置抽屉骨架；随后接入服务端 settings。
3. 按会话实现未读完成状态，覆盖同时运行、后台完成、失败、刷新恢复和查看后清除。
4. 实现桌面 rail、最近会话/agent flyout 和移动端既有导航兼容。
5. 最后实现头像上传、服务端验证/重编码、读取和恢复默认。

每一批都应覆盖 Firefox 和 Chromium 桌面端，并保留现有 760px 小屏流程。主题批次还需检查代码高亮、KaTeX、工具块、错误状态、焦点、滚动条和原生表单，不能只验收主页面背景。
